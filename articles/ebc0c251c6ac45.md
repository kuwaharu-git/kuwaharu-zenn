# LiveFxを支える技術スタックと設計上の工夫

## はじめに

この記事では、ライブパフォーマンス向け双方向システム「LiveFx」で採用している技術スタックと、設計上の工夫をまとめます。

LiveFxがどんなものかに関しては[LPサイト](https://livefx.siw.ac.jp/)がありますのでこちらを見ていただくのが一番わかりやすいと思います。

## システム概要
LiveFx は大きく分けて、以下の4つの要素で構成されています。

- control-pc
- front-smartphone
- back-smartphone
- infra
### control-pc
control-pc は、ライブ演出の司令塔となるコンポーネントです。コントロール用のフロントエンド画面を持ちながら、同時にバックエンドとしても動作し、スマートフォン側や外部機器との連携を担当します。

ここでは演出の切り替え、状態管理、MIDI コントローラーとの連携、Socket.IO を通じたリアルタイム配信などを担っています。

### front-smartphone
front-smartphone は、来場者や参加者のスマートフォン上で動くフロントエンドです。演出に応じた画面表示やエフェクト表示を担当し、リアルタイムに状態を受け取って表示を切り替えます。

### back-smartphone
back-smartphone は、スマートフォン向けのバックエンドです。HTTP API と Socket.IO による通信を提供しつつ、control-pc と TCP で連携します。スマートフォン側の接続をさばきながら、演出状態の同期やイベント通知の橋渡しを行います。

### infra
infra には、AWS 上で LiveFX を運用するための CloudFormation テンプレートや各種スクリプト、ドキュメントをまとめています。ECS Fargate、ALB、Route 53、CloudWatch などを使って、環境ごとのデプロイと運用を支えています。
